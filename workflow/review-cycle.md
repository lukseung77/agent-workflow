# Review Cycle Protocol

The review cycle is the repeating loop between a draft and a consolidated update.
This protocol standardizes the steps so the user's only action is starting each agent and pasting one prompt.

---

## The Cycle

```
Lead drafts → posts review-request → user pastes consultant prompt → consultants review
→ user pastes consolidation prompt → lead consolidates → repeat or advance stage
```

---

## Step 1: Lead Posts Review Request

After completing a draft (exploration file, proposal section, or any source-of-truth update), the lead agent posts a `review-request` item in its own collaboration file:

```md
### [XX-NNN] Review Request — [Topic]
- `date`: YYYY-MM-DD
- `from`: [lead]
- `to`: all
- `type`: review-request
- `status`: open
- `scope`: [file path(s)]
- `why`: [one sentence — what was drafted and why review is needed]
- `request`: Read the listed file(s) and leave feedback in your own collaboration file referencing this ID.
- `refs`: [file path(s)]
```

Immediately after posting, the lead generates a **ready-to-paste prompt** for each consultant and one for its own consolidation step:

---

### Ready-To-Paste Prompts (generated by lead after posting review-request)

**For each consultant agent** (paste this into the consultant's session):

```
There is a pending review request in the project.

1. Read `docs/collaboration-[lead].md` and find the open `review-request` item [XX-NNN].
2. Read the file(s) listed in its `refs` field.
3. Leave your feedback as a `review` item in your own collaboration file (`docs/collaboration-[yourname].md`), referencing [XX-NNN] in the `refs` field.
4. Update your index file.

Use the standard message format from `docs/collaboration-format.md`.
```

**For the lead's consolidation step** (paste this after all consultants have responded):

```
Consolidate reviews for [XX-NNN].

1. Read `docs/collaboration-[consultant-a].md` and `docs/collaboration-[consultant-b].md` for review items referencing [XX-NNN].
2. For each piece of feedback: accept, partially accept, or reject with a short rationale.
3. Integrate accepted feedback into the source-of-truth file(s) listed in [XX-NNN].
4. Post a `status` item in your collaboration file summarising what changed.
5. Mark [XX-NNN] as `done` and update your index.
```

---

## Step 2: Consultants Review

When a consultant agent starts a session, before doing anything else, check for pending review requests:

1. Read `docs/collaboration-index-[lead].md`
2. Look for any row with `type: review-request` and `status: open`
3. If found: that is your first task — read the referenced files and post your review

If no pending review requests exist, proceed with your own open items.

Consultant review item format:

```md
### [YY-NNN] Review — [XX-NNN] [Topic]
- `date`: YYYY-MM-DD
- `from`: [consultant]
- `to`: [lead]
- `type`: review
- `status`: done
- `refs`: [XX-NNN], [file path(s) reviewed]

Findings:
- [finding 1]
- [finding 2]
- ...
```

---

## Step 3: Lead Consolidates

When the user pastes the consolidation prompt, the lead:

1. Reads all consultant collaboration files for review items referencing the review-request ID
2. For each finding: decides accept / partially accept / reject, logs rationale
3. Applies accepted changes to source-of-truth files
4. Posts a consolidation summary `status` item in its collaboration file
5. Marks the original `review-request` item `done`

Consolidation summary format:

```md
### [XX-NNN+1] Consolidation — [XX-NNN]
- `date`: YYYY-MM-DD
- `from`: [lead]
- `to`: all
- `type`: status
- `status`: done
- `refs`: [XX-NNN], [all consultant review IDs]

Accepted:
- [finding] from [agent] → [what changed in which file]

Partially accepted:
- [finding] from [agent] → [what changed, what was excluded and why]

Rejected:
- [finding] from [agent] → [reason]
```

---

## Triggering The Next Cycle

After consolidation, if more drafting is needed:

- Lead continues drafting, then posts another `review-request`
- Consultants check for it on next session start
- Repeat

After consolidation, if the draft is complete:

- Lead posts a stage-readiness `status` item
- Consultants concur (or raise blockers) on next session
- User confirms stage transition

---

## User's Reduced Action Set

With this protocol, the user's only actions during a review cycle are:

| Action | What user does |
|--------|---------------|
| Trigger consultant review | Start consultant session, paste the prompt generated by the lead |
| Trigger consolidation | Start lead session, paste the consolidation prompt generated by the lead |

The lead generates both prompts automatically after posting the review-request.
The consultants check for pending requests automatically on session start.
The user does not need to explain context, scope, or format to any agent.
